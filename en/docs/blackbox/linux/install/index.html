Install Blackbox On Linux

Step 1: Install Docker

1Album Blackbox is distributed as a Docker Image. Your system must have Docker installed before you can install blackbox. The installation procedure varies among Linux distributions, please reference the official Docker Engine installation guide for details. On Ubuntu 16.04, the easist way to install Docker is:

$ sudo apt-get install docker.io

Step 2: Install Blackbox

You can download 1Album Blackbox with the following command:

$ docker pull 1album/blackbox

Downloading:

After download is completed, use docker images to list downloaded images:

Step 3: First Launch

Use the following command to launch 1Album Blackbox for the first time:

$ docker run -itd \
         -v /path/to/data:/data \
         --name mybox \
         1album/blackbox

The most important argument is -v, it links /data volume to local folder /path/to/data on your system. 1Album Blackbox stores everything in /data volume. You must link it to a local folder before starting the blackbox. Please do not manually alter the content in this folder, otherwise your data inside the blackbox may be damaged or lost.

Besides /data, there are three optional volumes that can be configured. /export is for exporting data out of the blackbox, and /import is for importing data back in. /cache is for storing temporarily files such as thumbnail images. The example below links all of them to local folders:

$ docker run -itd \
         -v /path/to/data:/data \
         -v /path/to/cache:/cache \
         -v /path/to/import:/import \
         -v /path/to/export:/export \
         --name mybox \
         1album/blackbox

After the docker run command, you can use docker ps command to list the running Docker containers. The last colume shows the name you specified with the --name argument previously.

Step 4: Link Account

You must add the blackbox to your account to start using it. First, obtain the Device Key from the log using the following command. mybox is what you specified with the --name argument previously:

$ docker logs mybox
