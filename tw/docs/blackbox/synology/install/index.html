在 Synology NAS 上安裝黑盒子

第一步：安裝 Docker

黑盒子是以 Docker Image 的方式發行，在安裝黑盒子前，必先在 Synology 機器上安裝 Docker。許多 Synology 的 NAS 機型都能支援，但似乎限於型號字尾有 “+” 的機型。完整支援的機型列表可參考 Synology Docker Package 頁面的最下面。

首先登入 Synology NAS 設備，打開桌面上的 Package Center 並搜尋 Docker，找到後直接按 Install 來安裝。如果找不到，表示您的機型不支援 Docker：

安裝成功後，Docker 會出現在已安裝 Package 列表中：

現在打開左上角的主功能表，裡面就會有 Docker 的圖示，點擊打開：

下面是 Docker 執行在 Synology NAS 上的主畫面：

第二步：安裝黑盒子

首先將左邊切換到 Registry 頁面，在上面的搜尋框裡輸入 1album 來搜尋 1Album 黑盒子：

選擇找到的項目，然後按 Download 鈕下載

下載中，右邊資料庫形狀的小圖示呈現動畫：

當資料庫小圖示不再有動畫，且會收到系統通知後，表示下載完成：

第三步：第一次啟動

選擇黑盒子的 Docker Image 後，按 Launch 按鈕啟動：

Launch 按下後跳出下面的對話框。按下 Advanced Settings 鈕進入進階設定畫面：

下圖圈起的 Enable auto restart 功能，可以在黑盒子遇到錯誤而不正常結束時，自動重新啟動黑盒子。我們建議打開：

接著切換到 Volume 頁面。資料目錄 /data 是 1Album 黑盒子儲存所有資料的位置，必須將 /data 指向 NAS 的檔案夾才能啟動黑盒子。按 Add Folder 鈕來設定：

選擇任何檔案夾，或按 Create Folder 建立新的檔案夾。這個檔案夾內的資料必須由黑盒子全權控制，請不要手動改變內容，否則可能會造成資料遺失或錯誤。選好後按 Select 鈕：

選定資料夾後，要在 Mount path 格子內填入 /data，必須全部小寫。這樣就設定好 /data 目錄：

除了 /data 外，還有三個額外的目錄可以設定，這些沒設也沒關係。/export 用來將黑盒子的內部資料輸出成一般檔案，/import 用來輸入外部資料進黑盒子，而 /cache 用來儲存如照片縮小圖等的暫存檔。下圖例子中，所有目錄都指定到 NAS 檔案夾。設定好之後按 OK 鈕：

回到啟動設定畫面，按 Next 鈕：

再次確認至少 /data 有設定連結的檔案夾，按 Apply 鈕來儲存設定：

第四步：連結帳號

必須把黑盒子加到您的帳號中，才能開始使用。首先左邊切換到 Container 頁面，找到執行中的黑盒子，按下 Details 鈕：

Detail 對話框打開後，切換到 Log 頁面，其中箭頭所指的地方就是黑盒子的 Device Key：

打開 1Album app，如下圖所示，將 Device Key 輸入：

現在開始，這個黑盒子就會為您工作了。同時，在黑盒子端會顯示您登入的帳號名稱。由於這個 Log 頁面不會自動更新，您需要按一下右下角小小的更新鈕才會看得到：
